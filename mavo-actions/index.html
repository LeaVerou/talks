Data mutation functions

We kept these functions minimal, to delegate selection and filtering logic to Mavo expressions. This maximizes the amount of computation specified in a reactive fashion.

  •  set(reference, value)
  • delete(ref1, ref2, ref3, ...)
  • add(collection, data, index)
  • move(from, to, index)

    Conclusions
  • Novices seem to be able to formulate data actions and find them useful
  • Syntax rigidity is not natural. Yay sloppiness?
  • Repetition is not natural. Yay heuristics?
