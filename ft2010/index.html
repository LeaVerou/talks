Isn’t CSS2.1 sufficient?

Question: What bugs you the most when coding CSS (2.1) today?

IMHO:

  • Decorative images
  • Markup clutter
  • JavaScript for presentational stuff

CSS3 attempts to solve these problems

Problem #1

Needing decorative images

Why is it a problem?
  • Maintenance hell
  • Consumes more bandwidth
  • More http requests ⇒ slower
  • Style harder to understand
  • Users get to see the page without them

Appendix

Dealing with all these browsers

Why bother?
  • By all means don’t, if it’s a personal fun project! ☺
  • Clients care though
  • Sad but true: A large percentage of internet users use crappy browsers, we can’t ignore them just yet
  • Even if you choose not to, let users know
Graceful degradation
in a nutshell

Browsers ignore stuff they don’t understand

That includes:

  • Unknown CSS properties and values
  • CSS rules with a selector they can’t parse

If a CSS property is defined twice in the same rule, only the last one counts

Graceful degradation
in CSS values
background: black;
background: rgba(0, 0, 0, .75);
  • If RGBA is supported, the second declaration overrides the first
  • Otherwise, it’s ignored and the first one is applied
Dealing with
-vendor-prefixes

Just include all of them, browsers will pick the one they understand

-moz-transition: 1s ease-in;
-webkit-transition: 1s ease-in;
-o-transition: 1s ease-in;
transition: 1s ease-in;

Put the prefix-less version last

Detecting CSS
property support
in JavaScript
function isPropertySupported(property) {
if(property in document.body.style) return true;
    
var prefixes = ['Moz', 'Webkit', 'O', 'ms', 'Khtml'],
	prefProperty = property.charAt(0).toUpperCase() + property.substr(1);
	
for(var i=0; i<prefixes.length; i++) {
	if((prefixes[i] + prefProperty) in document.body.style) return true;
}

return false;
}
Detecting CSS
value support
in JavaScript
function isRGBASupported() {
var element = document.createElement('div');
    
try { element.style.color = 'rgba(0,0,0,.5)'; }
catch(e) { return false; }

return !!element.style.color;
}
Detecting CSS3
selector support
in JavaScript
function isSelectorSupported(selector) {
try {
   document.querySelector(selector);
   return true;
} catch(e) { return false; }
}
  • Assumes Selectors API support, so it’s only useful for CSS3 selector detection
  • Doesn't test for buggy support, just whether the browser recognizes the selector

Last but not least...
  • Don't believe the hype,
    study the specifications yourself

    • Selectors Level 3
    • Backgrounds and Borders Module Level 3
    • CSS 2D Transforms Module Level 3
    • CSS Transitions Module Level 3
    • CSS3 Generated and Replaced Content Module
